Git Hooks for Checking Tabs vs Spaces

by Josh Freeman    Apr 02, 2014    Software development Git, Githooks, Source control

The Problem: Tabs in Source Control

Here's a recent conversation I had:

  • Me: PSR Standards...blah, blah, blah...no tabs...blah, blah, blah...indent 4 spaces for PHP!
  • Boss: OK, that's what we'll do

A few days later

  • Me: Sent a pull request
  • Boss: Okay great! Please replace all tabs with spaces. We agreed to use 4 spaces for indention, not tabs.

It's time to figure out what's going on.

~/path/to/project$ grep -r -P '\t' .

Uh, oh...

The Answer: Git Hooks

While wars rage on about why to use spaces or tabs over the other, we are not here to talk about why we do not use tabs in a project. No, no, we are here to talk about git hooks.

In the lifecycle of a project controlled through Git, there are actions a user takes to manage their code, mostly adding, committing, and pushing to another repository. Git Hooks are a kind of event listener for these actions. Each hook is an executable script that fires off before, during, or after certain git events. For pre-action hooks, if the script exits with a non-zero status—Unix-speak for the script failed—the action is halted and will not continue. This is what we are going to rely upon.

Inside a Git respository, you'll find the hooks directory:

~/path/to/project/$ cd .git/hooks

Inside this directory, git provides several sample hooks to demonstrate their naming convention and functionality. To make any sample hook work, all you need to do is rename it and ensure it is executable (if it is not already):

~/path/to/project/.git/hooks$ mv pre-commit.sample pre-commit
~/path/to/project/.git/hooks$ chmod +x pre-commit

Now, the example files are written using bash, but you can use any language so long as you have an interpreter installed and can execute it. In my case, we'll be using Ruby.

I want my pre-commit hook to do three things: check the files I am about to commit for tab characters, list the locations of the tab characters, and stop the commit from happening so I can fix the problem.

I Have Pre-Committment Issues

Let's build our git hook, piece by piece. First, we need to declare that we are using Ruby to execute this file. All work is done in .git/hooks/pre-commit

#!/usr/bin/ruby
...

Now, it's time to get the list of files we are about to commit. Git has a handy command for just this purpose. git diff --cached --name-only. We'll git this list and map it to an array. We'll also initialize our offending_lines hash.

...
git_files = `git diff --cached --name-only`.lines.map(&:chomp)

offending_lines = {}
...

With the list of files in hand, we will check each file, line-by-line, for the tab character. Two quick gotchas: we need to ensure that the files we are checking are actually files and text-based (not images, archives, etc.). Without including a library, we can check like this:

...
# https://www.ruby-forum.com/topic/122170#544763

def File.binary? name
  open name do |f|
    while (b=f.read(256)) do
      return true if b[ "\0"]
    end
  end
  false
end

git_files.each do |file|

  next unless File.file?(file)
  next if File.binary?(file)

  ...

end
...

Let's find those elusive tabs. We just iterate through each line of each staged file, check for a tab character, and add it to the offending_lines hash if found.

...
git_files.each do |file|

  next unless File.file?(file)
  next if File.binary?(file)

  File.open(file) do |f|
    ln = 1
    lna = Array.new

    f.each_line do |line|

      lna.push ln  if line.match /\t/

      offending_lines[file] = lna unless lna.empty?

      ln += 1
    end
  f.close
  end
end
...

If we found any tabs, we need to output the results and exit with a non-zero status.

...
unless offending_lines.empty?

  puts "\n"
  puts "Pre-Commit Hook Failure (.git/hooks/pre-commit)"
  puts "Tab characters are illegal in source control and Tabs were found in the following files"
  puts "\n\n"

  offending_lines.each do |file, line_numbers|
    puts file
    line_numbers.each_slice(15).to_a.each { |sub_array| puts "\t" + sub_array.join(', ') }
    puts "\n"
  end

exit 1
end

Let us put it all together:

.git/hooks/pre-commit

#!/usr/bin/ruby

# https://www.ruby-forum.com/topic/122170#544763

git_files = `git diff --cached --name-only`.lines.map(&:chomp)

offending_lines = {}

def File.binary? name
  open name do |f|
    while (b=f.read(256)) do
      return true if b[ "\0"]
    end
  end
  false
end

git_files.each do |file|

  next unless File.file?(file)
  next if File.binary?(file)

  File.open(file) do |f|
    ln = 1
    lna = Array.new

    f.each_line do |line|

      lna.push ln  if line.match /\t/

      offending_lines[file] = lna unless lna.empty?

      ln += 1
    end
  f.close
  end
end

unless offending_lines.empty?

  puts "\n"
  puts "Pre-Commit Hook Failure (.git/hooks/pre-commit)"
  puts "Tab characters are illegal in source control and Tabs were found in the following files"
  puts "\n\n"

  offending_lines.each do |file, line_numbers|
    puts file
    line_numbers.each_slice(15).to_a.each { |sub_array| puts "\t" + sub_array.join(', ') }
    puts "\n"
  end
exit 1
end

Now, throw some tab characters into you're project (a test branch, please). And try to commit the changes.

~/path/to/project$ git add posts/githooks.md
~/path/to/project$ git commit -m 'Attempted commit with tab characters'

Tab characters are illegal in source control and Tabs were found in the following files


posts/githooks.md
    50, 51, 55

Conclusion

Git Hooks are awesome. While this post uses them to prevent tabs sneaking into source control, you can use them to do just about anything. If you'd like to check out this particular hook, check out my gist over on Github.

P.S. By the way, I forgot to fix my editor back to spaces before I committed. It really does work.

$ gitcommit "Tutorial complete, requires proofreading"
