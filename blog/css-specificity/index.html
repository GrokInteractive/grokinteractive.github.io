CSS Specificity

by Josh Freeman    Apr 05, 2014    Training Css

About a year ago, I had a front-end project that was a mess, particularly the CSS. Selectors were often 6 or more selectors long and had !important appended to properties over the code base—style sheets, style tags, inline styles; if you could override a style and make it important, rest assured, it was.

It was pretty clear that the previous developer either did not understand CSS very well or could not care less (probably a little of both).

// Do not do this. Do not ever do this. (Unless you have to do this.)

body.ie  #mainheader nav ul.nav li.nav-link.active a * {
    color: black !important;
}

I had another challenge: I could only change the style sheets; I was prohibitied from touching the markup or the JavaScript that gave elements styling directly. This presented a huge problem, especially for the important inline styles and it all boiled down to CSS specificity as both the problem and solution.

What is Specificity?

Say we have a style sheet defines a style for an element, a style tag that has a conflicting property value for the same element, and an inline style that also has a conflicting property value for the same element.

<!-- styles.css -->
p {
    color: black;
}

<!-- index.html -->
<style>
p {
    color: green;
}
</style>

<p style="color:red;">Header</p>

We're telling the browser three different things, so which one wins? The one with the highest specificity. Specificity is applied at the property level, so if multiple style sheets all declare styles for different properties on the same element, all the property declarations will be applied.

How to Calculate Specificity

This is where specificity comes into play. There are some pretty basic rules to follow that are best visualized. First let's 'para-quote' the WrC:

  • count 1 if the property value is declared !important
  • count 1 if the declaration is from is a 'style' attribute rather than a rule with a selector, 0 otherwise
  • count the number of ID selectors in the selector
  • count the number of class selectors, attributes selectors, and pseudo-classes in the selector
  • count the number of type selectors and pseudo-elements in the selector
  • ignore the universal selector

W3C - Calculating a Selector's Specificity 2011 W3C - Calculating a Selector's Specificity 2009

  • Treat the five rules below as five digits in a number with no maximum. -- If for some awful reason you have 30 class selectors, your class count will be 30. -- An interesting CSS trick, you can concatenate a selector multiple times to increase specificity.
  • We take a selector and count up each type of selector within according to rules above.
  • The count of each type is compared with other declarations, per rule.
  • If multiple selectors have the same specificity, the last one declared wins.

A Quick Word on !important

Now is a good time to discuss what is really !important. Declaring a property !important is an quick and easy way to override a selector you cannot modify. Be careful with this approach, however; once you decalare a selector's property as !important, the only way to then override it is to again declare that property as !important with a selector that has higher specificity.

p {
    color: black;
}

p {
    color: red !important;
}

body p {
    color: blue !important;
}

p.green {
    color: green !important;
}

Try to avoid getting into the selector rat race in this way. It hurts your browser's performance in several ways and becomes much more difficult to maintain.

This brings me to a very important point regarding CSS:

If everything is important, nothing is important.

Final Thoughts

Hopefully, you a better understanding of how CSS specficity works. We've only touched upon the mechanics of CSS selector rules—we have not addressed CSS selector performance. For now, it is enough to understand that a 6 selector long selector is very bad for many reasons.

Ideally, you would go back to the orignal declaration and change it, but sometimes, that's not an option. If you ever need to override a CSS framework or fix a legacy project, this will definitely come in handy.
