Setting Up a PHPUnit Testing Environment

by Josh Freeman    Mar 21, 2014    Software development, Testing Php, Phpunit

You should be testing. You know you should be testing. But you are not testing. Why?

More often than not, it is probably because you have spent more time debugging your testing environment than your application and you've lost all motivation to continue. That was me for quite a while.

It's time to stop this tomfoolery and test your application like your better senses are telling you. This post is going to focus specifically on PHP and unit testing with PHPUnit. Unit testing is one piece of the testing puzzle, and if you're designing your components SOLIDly, testing should become an asset to your development NOT a hindrance.

Installing Composer

To get started, we are going to create a project and treat PHPUnit as a project dependency. I have found that the easiest way to manage your project dependencies is through Composer. You can install this one of two ways, globally on your system, or locally within your project. Since we're new and just want to play with composer, we will opt for the local method. At the very least, you will need PHP 5.3.2 or above to use composer.

# Create your phpunit-example
~$ mkdir -p phpunit-example/src/PHPUnitExample
~$ cd phpunit-example

# Here we download the composer installer and execute it with PHP
~/phpunit-example$ curl -sS https://getcomposer.org/installer | php

# Now we initialize this project as a composer project, this will create your composer.json
~/phpunit-example$ php composer.phar init

Installing PHPUnit

Now that we have the project initially setup, we are going to create our composer.json. This file is the manifest for your project, describing itself and the dependencies it requires. Running php composer.phar init will start an interactive shell to generate our composer.json.

The Package name is broken into two parts, the vendor and the project name. Use your name or your organization name as the vendor. Here, I will be using my GitHub name.

~/phpunit-example$ php composer.phar init
This command will guide you through creating your composer.json config.

Package name (<vendor>/<name>) [hostname/phpunit-example]: jdfreeman/phpunit-example
Description []: This project demonstrates how to quickly setup phpunit
Author: Josh Freeman <joshf@grok-interactive.com>
Minimum Stability []: stable
License []: MIT

Define your dependencies.

Would you like to define your dependencies (require) interactively [yes]? no
Would you like to define your dev dependencies (require-dev) interactively [yes]? no

...

Do you confirm generation [yes]?

We are not done yet, instead of auto-generating the dependencies, let us learn how to manually add dependencies. Open your composer.json and you should see something similar to below:

{
    "name": "jdfreeman/phpunit-example",
    "description": "This project demonstrates how to quickly setup phpunit",
    "license": "MIT",
    "authors": [
        {
            "name": "Josh Freeman",
            "email": "joshf@grok-interactive.com"
        }
    ],
    "minimum-stability": "stable",
    "require": {

    }
}

Within your require block, we're going to ensure you have a sufficiently up-to-date version of PHP.

{
    ...
    "require": {
        "php": ">=5.4.0"
    },
    ...
}

Now, we are going to add PHPUnit to the list of dependencies. Under the require block, append the following:

{
    ...
    "require-dev": {
        "phpunit/phpunit":"4.0.*"
    },
}

Finally, we are going to permit the autoloading of assets through namespacing. { ... "autoload": { "classmap": [ "src/Project" ] } } Save your composer.json and ensure it validates through php composer.phar validate. Pay attention to any missing commas. Your final composer.json should look similar to the following:

composer.json

{
    "name": "jdfreeman/phpunit-example",
    "description": "This project demonstrates how to quickly setup phpunit",
    "license": "MIT",
    "authors": [
        {
            "name": "Josh Freeman",
            "email": "joshf@grok-interactive.com"
        }
    ],
    "minimum-stability": "stable",
    "require": {
        "php": ">=5.4.0"
    },
    "require-dev": {
        "phpunit/phpunit":"4.0.*"
    },
    "autoload": {
       "classmap": [
            "src/PHPUnitExample"
        ]
    }
}

Now it's time to install PHPUnit. Since it's already defined in our composer.json, we simple need to run php composer.phar install

At this point, a new directory vendor/ should have been created and all the packages required to run PHPUnit should have been installed. There should also be a composer.lock file which describes the current state of all your dependencies.

Configuring PHPUnit

Now that we have PHPUnit installed locally, we will create the test directory and configuration file PHPUnit uses to run the tests.

~/phpunit-example$ mkdir tests
~/phpunit-example$ touch phpunit.xml

Open your phpunit.xml and write the following:

phpunit.xml

<?xml version="1.0" encoding="UTF-8"?>
<phpunit backupGlobals="false"
         backupStaticAttributes="false"
         bootstrap="./vendor/autoload.php"
         colors="true"
         convertErrorsToExceptions="true"
         convertNoticesToExceptions="true"
         convertWarningsToExceptions="true"
         processIsolation="false"
         stopOnFailure="false"
         syntaxCheck="false"
>
    <testsuites>
        <testsuite name="Application Test Suite">
            <directory>./tests/</directory>
        </testsuite>
    </testsuites>
</phpunit>

With our phpunit.xml file updated, we are ready to start writing tests!

Testing Time

Now that we can write tests, we are going to build a simple calculator through TDD (Test-Driven Development). Using the TDD paradigm, we will write our test first, ensure it fails, then write the necessary code to make it pass, and then refactor as needed.

We will begin by creating two files: the class to be tested and the class holding the tests.

~/phpunit-example$ touch src/PHPUnitExample/Calculator.php
~/phpunit-example$ touch tests/PHPUnitExample/CalculatorTest.php

In your tests/PHPUnitExample/CalculatorTest.php, write the following:

tests/PHPUnitExample/CalculatorTest.php

<?php

class CalculatorTest extends PHPUnit_Framework_TestCase
{
    public function testAdd()
    {
        $calculator = new PHPUnitExample\Calculator;

        $result = $calculator->add(3, 4);

        $this->assertEquals(7, $result);
    }
}

In your src/PHPUnitExample/Calculator.php, write the following:

src/PHPUnitExample/Calculator.php

<?php

namespace PHPUnitExample;

class Calculator
{
    public function add()
    {
    }
}

Now that you have the minimum files needed to run test, we need to update the autoloader that will allow PHPUnit to require the needed files automatically.

# updates the vendor/autoload.php, which is required in the phpunit.xml
~/phpunit-example$ php composer.phar dumpautoload

It is time to actually run our first test! This test should fail, since we have defined the method, but not built any functionality.

~/phpunit-example$ php vendor/bin/phpunit
PHPUnit 4.0.12 by Sebastian Bergmann.

Configuration read from /vagrant/sites/phpunit-example/phpunit.xml

F

Time: 575 ms, Memory: 3.50Mb

There was 1 failure:

1) CalculatorTest::testAdd
Failed asserting that null matches expected 7.

/vagrant/sites/phpunit-example/tests/PHPUnitExample/CalculatorTest.php:11

FAILURES!
Tests: 1, Assertions: 1, Failures: 1.

As expected, our test failed; now it is time to actually make this test pass. Return to your Calculator class and write the following:

public function add($a, $b)
{
    return $a + $b;
}

Return to your terminal and run the tests again.

~/phpunit-example$ php vendor/bin/phpunit
PHPUnit 4.0.12 by Sebastian Bergmann.

Configuration read from /vagrant/sites/phpunit-example/phpunit.xml

.

Time: 579 ms, Memory: 3.25Mb

OK (1 test, 1 assertion)

Voila! Now we have our testing environment set up and our first passing test! Now it is time for you start reading PHPUnit Documentation and learn all that PHPUnit can bring to the table.
