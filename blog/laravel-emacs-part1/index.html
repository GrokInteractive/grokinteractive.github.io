Laravel & Emacs - Part 1

by Chris Hardee    Mar 23, 2014    Environments Emacs

I have been using Emacs for development for four years now, it initially started when I started learning Linux sysadmin work and working from a terminal. I quickly got frustrated with the limitations, and embarrassed from on-lookers, when I would load up Nano for text editing. I got comfortable enough with vi to do some simple things and started learning regular expressions. The time soon came to learn something I can start calling my own that could handle much more than what vi was initially setup for.

Googling for something akin to "What editor should I learn after vi?" and you will run amuck with suggestions for using Vim. But instead of that you should learn Emacs. Really now, you should learn Vim and just Deal With It. Then there is a picture of Richard M. Stallman in a christlike outfit, devoted to The Church of Emacs.

Those of you, in the gentle comfort of Sublime or Textmate might mercifully be spared the long-running editor wars waged between Emacs and Vim. They more or less boil down to different philosophies on what constitutes an editor, and that is all that really needs to be said on that subject.

I played around with both for a few days and went with Emacs.

Discovering web-mode

At Grok Interactive we have been doing a lot of Laravel work, mostly custom applications for very specific needs. Working in Emacs the first thing that hit me was how little the default php-mode fitted my needs when it came to working in php files in general, but especially .blade. templates.

After some research I came across web-mode, and it suited my purposes neatly and I now use it for several templating engines (including Rails .erb files).

I hope to document my usage of hacking laravel in Emacs in several parts. I use standard Emacs command conventions in this, please refer to basic Emacs documentation if you're confused.

Part 1 - Configuring Emacs to use web-mode for laravel

If you are using Emacs 24 or later you can grab it off the melpa repository. To enable this or any other of the various excellent unofficial elisp repositories you simple need to add this to your configuration:

(setq package-archives '(("gnu" . "http://elpa.gnu.org/packages/")
                       ("melpa" . "http://melpa.milkbox.net/packages/")))

Adding this to your configuration will slow things down a bit when you got to list available packages:

M-x package-list-packages

You'll be rewarded though with tons of available packages, search for web-mode:

C-s web-mode

With your cursor on it, hit i to mark it for installation then x to execute the actions (you can browse around for other kewl packages too and mark them for installation, hit x when you're done). Emacs will ask you if you want to install these packages, type yes and hit enter. It'll download and compile the elisp. Ok you've got web-mode!

Let's set it up, open up your config file or switch to the buffer if you already have it open. First we need to load web-mode:

(require 'web-mode)

Then we set it to switch to web-mode whenever we open up a .php (.phtml) file or a .blade file:

(add-to-list 'auto-mode-alist '("\\.php\\'" . web-mode))
(setq web-mode-engines-alist
  '(("php"    . "\\.phtml\\'")
    ("blade"  . "\\.blade\\."))
)

This is also a good place to set any indentation levels or tabs-to-spaces settings:

(setq-default indent-tabs-mode nil)
(setq web-mode-code-indent-offset 4)
(setq web-mode-indent-style 4)

Reload your buffer if needed M-x eval-buffer then you're good to go.

That is it for part one, you should be ready to tackle any .blade or .php files. Next time we'll go over efficiently coding and formatting in web-mode.
